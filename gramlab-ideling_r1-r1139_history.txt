1) Input files

Use parameters in MailItemCollectionReader.xml:
 - InputDirectory: specify the path to the input directory where mails are stored as *.xml 
 - InputUrl: (optional) specify the URL where to fetch the *.xml input files (if not empty
	this parameter preempts InputDirectory)
 - Encoding: currently ISO-8859-1
 - Language: currently not in use (always French)

In case we process local files, the input files are renamed appending ".processing" while
they are being processed, and renamed with suffix ".done" when finished.

2) Output files

Use paremeters in XmiWriterCasConsumer.xml:
 - OutputDirectory: (mandatory) the path to the output directory
 - OutputURL: (optional) the URL where to post the output files

When the local output files are complete, a semaphore file with the same name and
ending with ".ok" is generated.

If the OutputURL parameter is not empty, the output file is posted to this URL.

Initial import.

Simple CPE architecture with:
- MailItemCollectionReader: reads XML mail dumps, the mail body text will become the CAS document text, and all other nodes become document annotations;
- RegExAnnotator: reused from UIMA sandbox, with a concept file allowing to recognize email addresses and URLs;
- DynamicDictionaryAnnotator: takes the recognized emails and urls and create a dynamic dictionary for all emails and one for all urls using DELAF format;
- UnitexAnnotator: wraps the calls to the native Unitex
